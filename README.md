## 지원 사유

처음으로 입사한 회사에서 서비스 운영업무를 맡았고, 주로 Java와 Servlet을 이용해 API, VOD 서비스의 회선, 공지, 광고, 통계, 모니터링 페이지, 시스템 등을 개발하며 차근차근 실력을 쌓았습니다. 
프로그래밍이 어느 정도 손에 익은 2016년부턴 저에게 맞는 개발은 무엇인지 연구하기 위해 많은 시도를 했습니다. 지인들에게 다양한 업무를 받아 소화했으며, 유니티 전문과 과정을 수강하기도 했고, 앱과 서비스를 개발해내는 성과도 얻었습니다.
아직도 성장하고 싶고 사람들에게 편의성을 제공하는 서비스, 누군가 제가 만든 프로그램을 잘 사용하고 있다는 소식만 들어도 행복한 개발자를 꿈꾸고 있으며 실력있는 개발자분들과 문화가 있는 곳에서 일하고싶습니다.

#### 업무 스타일

개발을 하는 것을 좋아하고 제가 개발한 것들을 공유하고 토론하는 것을 좋아합니다. 
그렇기 때문에 항상 가독성이 높은 코드, 의미가 명확한 코드를 지향하고 이를 유지하기 위해서 제가 일하는 것과 관련된 것들을 학습하며 개선해나갑니다. 
이전 직장에서는 처음 사용한 Node의 코드를 개선하기 위해서 Javascript 스터디를 쫒아다니고 개선된 내용을 제 작업물에 최대한 녹여내려고 노력하였습니다. 

#### 직무 수행 경험 및 자세

저는 웹개발자로써 매번 다른 환경에서 일했습니다.  회사에서는 Java기반의 웹 어플리케이션을 다음은 php, laravel 기반의 환경이었고 가장 최근 이력은 Node기반의 웹 어플리케이션 개발이었습니다.
항상 새로운 환경에 노출되어 있었지만 그 것을 두려워하지 않았던 이유는 항상 저의 포지션은 웹 API 개발 및 관리페이지의 개발이었습니다. 이는 개발화는 환경만 달라졌을 뿐 제가 했던 일은 서비스의 관리를 담당하는 관리페이지 개발에 있습니다.
첫 시작은 Ntv라는 서비스의 관리페이지를 만들면서 시작하였습니다. 셋톱박스의 개통 및 데이터 변경부터 서비스에 들어가는 공지, 광고 등등 제가 만든 웹 어플리케이션으로 서비스가 관리 할 수 있는 서비스를 개발했습니다.
